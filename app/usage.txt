Cryptography Application -- Usage

If this project is being forked, build with CMakeLists. Then you may
find an executable in build/Debug. Run the application.

When the app is open, it can only perform one task. If second performance 
is attempted, the application will crash.
Don't worry. the application can be run again to perform other tasks.

In file encryption/decryption, when the empty option is chosen, it will 
default to "decrypt" action.

There are console commands for further details of tasks performed in 
the application.

If the application can be run on cmd/powershell

> cd /d "root_of_project"
> cd app
(if error occurs or to remove cmake cache, else ignore)
> rmdir /s /q build 

> cmake -S . -B build -DCMAKE_PREFIX_PATH="C:/Qt/6.10.2/msvc2022_64" (path to qt default installation)
> cmake --build build
> cd build/Debug
> "C:\Qt\6.10.2\msvc2022_64\bin\windeployqt.exe" --qmldir ..\.. Cypherin.exe
> Cypherin.exe
